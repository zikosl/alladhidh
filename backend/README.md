# Restaurant POS Backend

TypeScript Express backend for the restaurant POS frontend, powered by PostgreSQL, Prisma, and stock movements.

## Features

- Product catalog with category, icon, and color metadata for the fast POS grid
- Transactional order creation for dine-in, take-away, and delivery
- Kitchen workflow endpoints for `pending`, `preparing`, and `ready`
- Cashier payment flow with `cash` and `card`
- Delivery status tracking
- Stock derived only from `stock_movements`
- Dashboard metrics and sales reporting

## Setup

1. Create a PostgreSQL database named `restaurant_pos`, or use the provided Docker Compose stack from the parent folder.
2. Copy `.env.example` to `.env` and update DB credentials.
3. Install dependencies:

```bash
npm install
```

4. Generate the Prisma client:

```bash
npm run prisma:generate
```

5. Push the Prisma schema and prepare database objects:

```bash
npm run db:push
npm run db:prepare
```

6. Seed demo data:

```bash
npm run db:seed
```

7. Start the server:

```bash
npm run dev
```

8. Build production output:

```bash
npm run build
```

- `GET /api/stock`

- `GET /api/reports/dashboard`

## Notes

- Stock is never stored as a static column.
- Every stock change is logged in `stock_movements`.
- Prisma is used as the backend data layer, with raw SQL reserved for stock/reporting aggregates where it keeps the code simpler.
- Seed data is generated by Prisma from [src/scripts/seed.ts](/Users/zakaria/Desktop/alladhidh/backend/src/scripts/seed.ts:1), not by raw SQL inserts.
- Order creation uses a DB transaction and validates recipe-driven stock availability before consuming ingredients.
- The root `docker-compose.yml` starts `backend`, `admin`, and `postgres` together.
